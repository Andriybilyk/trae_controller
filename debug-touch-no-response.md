# [OPEN] touch-no-response

## Симптом
- Після прошивки UI стартує, але екран не реагує на натискання.

## Відомі спостереження
- `DISPLAY: Touch initialized (XPT2046, mode=0, hz=250000, host=1)`
- `LVGL_BACKEND: LVGL UI initialized, entering main loop`
- У моніторі немає ознак обробки натискань.

## Гіпотези
1. XPT2046 не віддає точки через невдалий режим `IRQ`/polling або конфіг `esp_lcd_touch`.
2. Точки читаються драйвером, але не доходять до `lvgl_touch_read_cb()`.
3. `lv_indev` застрягає у скинутому/неактивному стані після попередніх правок навігації.
4. Координати/strength читаються, але debounce/фільтр завжди переводить стан у `RELEASED`.
5. Є гонка доступу до SPI/touch, і read callback отримує порожні дані.

## План
- Додати короткі runtime-логи в `touch_read_once()`, `display_driver_touch_probe()` і `lvgl_touch_read_cb()`.
- Прошити, відтворити проблему, зібрати логи.
- За логами підтвердити або відкинути гіпотези.

## Статус
- Очікується інструментування та повторна перевірка.
