## What This Is

This project was generated by [PicoPixel.io](https://picopixel.io), the ultimate
drag-and-drop visual UI builder for microcontrollers like ESP32 and other embedded devices
capable of running LVGL. PicoPixel gives hardware engineers a Figma-like design
experience — build your UI visually in the browser, then export production-ready C code
for your firmware.

This export targets **LVGL v8.x**. All generated code uses LVGL 8.x APIs — do not mix
with LVGL 9.x functions.

## Architecture

### Initialization Flow

```
Your main.c:
  1. lv_init()                    — Initialize LVGL library
  2. Set up display driver         — Register flush callback
  3. Set up input driver           — Register touch/button read callback (optional)
  4. ui_init()                     — Creates all screens, loads first screen
  5. Main loop:
       lv_timer_handler()          — LVGL internal processing
       ui_tick()                   — Per-frame update for current screen
```

### How Screens Work

- Each screen has `create_screen_*()`, `tick_screen_*()`, and `delete_screen_*()` functions
- `ui_init()` calls `create_screens()` which creates all screens at startup
- `loadScreen(SCREEN_ID_XXX)` switches screens with a 200ms fade animation
- `ui_tick()` dispatches to the current screen's tick function

### The `objects_t` Struct

All widget pointers live in a global `objects_t objects` struct. This lets you access any
widget from anywhere:

```c
// Access a widget directly
lv_label_set_text(objects.label_title, "Hello");
lv_obj_add_flag(objects.panel_menu, LV_OBJ_FLAG_HIDDEN);
```

Widget pointers are `NULL` until their screen's `create_screen_*()` runs, and are set back
to `NULL` when `delete_screen_*()` runs.

## Public API

### Entry Points (you must call these)

```c
void ui_init();                              // Call once after LVGL + display driver init
void ui_tick();                              // Call each frame after lv_timer_handler()
void loadScreen(enum ScreensEnum screenId);  // Navigate to a screen
```

### Screen IDs

```c
enum ScreensEnum {
    SCREEN_ID__2 = 1,
    SCREEN_ID__1 = 2
};
```

### Global State

```c
extern objects_t objects;          // All widget pointers
```

| Type | LVGL Function | Notes |
|------|--------------|-------|
| Bar | `lv_bar_create()` | Linear indicator with min, max, value |

## Key Patterns & Gotchas

- This project targets **`LV_COLOR_DEPTH = 32`** (ARGB8888). Set this in your `lv_conf.h`.
- **Enable fonts**: Every built-in Montserrat size listed above must be enabled in `lv_conf.h`, or you'll get linker errors.
- **Enable widgets**: Every widget type listed above must be enabled (`LV_USE_*`), or you'll get compilation errors.
- **Widget pointers are NULL before screen creation**: Don't access `objects.some_widget` before `ui_init()` runs.
- **This export targets LVGL v8.x**: Do not mix with LVGL 9.x APIs — the function signatures, enums, and style system are different.
- **`LV_MEM_SIZE`**: If you have many widgets or large images, increase `LV_MEM_SIZE` in `lv_conf.h` (default 48KB may not suffice).
