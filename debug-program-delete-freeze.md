[OPEN] Debug session: program-delete-freeze

## Симптом
- При підтвердженні видалення програми в поп-апі контролер зависає (UI перестає реагувати).

## Очікувана поведінка
- Після підтвердження: програма видаляється з памʼяті та NVS, список оновлюється без зависання.

## Гіпотези (фальсифіковані)
- H1: Зависання викликає краш/ресет (WDT/panic), але без видимого повідомлення на UI; підтвердиться логом з UART.
- H2: Зависання викликає некоректне видалення/перемальовка LVGL обʼєктів (видалення екрана/оверлею всередині callback) → assert/loop; підтвердиться логами LVGL/панікою.
- H3: Зависання викликає пошкодження памʼяті при зсуві масивів програм під час delete (копіювання з перекриттям) → panic або дивна поведінка; підтвердиться логами/бек-трейсом.
- H4: Зависання — це довге блокування на NVS commit під час UI callback, що блокує UI loop і тригерить watchdog; підтвердиться таймінгами та WDT у логах.

## План збору доказів
1) Запустити Debug Server (program-delete-freeze) і зʼєднати forwarder з COM11 → Debug Server.
2) Відтворити: Programs list → Delete → Confirm.
3) Зібрати UART логи до/після натискання Confirm і визначити, чи є panic/WDT/assert.
