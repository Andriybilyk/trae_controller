[OPEN] add-step-lag

# Симптом

- Кнопка `Додати крок` все ще реагує із великою затримкою.
- Користувач натискає кнопку, але новий крок з'являється помітно пізніше.

# Гіпотези

1. Подія натискання доходить до callback із затримкою ще на рівні touch/LVGL event dispatch.
2. `lv_async_call()` ставить додавання в чергу, але callback виконується лише через кілька проходів UI loop.
3. Найдовша затримка сидить усередині `screen_programs_refresh_rows()`.
4. Найдовша затримка сидить усередині `programs_update_info()`.
5. Найдовша затримка сидить у `screen_programs_scroll_to_last()` або в relayout після додавання.

# План

- Додати точкові часові логи в маршрут `PRESSED -> action_programs_add_step -> async callback -> refresh/info/scroll`.
- Зняти реальні часові дельти і визначити, де саме накопичується пауза.
- Після цього зробити мінімальний evidence-based fix.
