# Симптом

- На екрані `Графік програми` крива виглядає обрізаною/зазумленою.
- Початкова та/або кінцева частина кривої не потрапляє в межі контейнера так, як очікується.
- Останні логічні правки масштабу/відступів не змінили видимий результат на фото користувача.

# Гіпотези

1. На екран графіка потрапляє не повний набір кроків, а лише частина моделі програми.
2. `total_time_s` або `step_cnt` під час побудови графіка відрізняються від того, що реально бачить користувач у редакторі програми.
3. Координати точок обчислюються правильно, але під час рендеру `lv_line`/маркерів частина графіка кліпиться не там, де очікується.
4. Для `selected_program_id == 255` (`Нова програма`) графік бере не той набір даних або не той лічильник кроків.
5. Після побудови екрана додатковий layout/scaling змінює геометрію контейнерів, але не узгоджує її з точками лінії.

# План

- Додати точкові логи в маршрут відкриття `Графік програми`.
- Зафіксувати: `selected_program_id`, `step_cnt`, `total_time_s`, `y_max`, `panel/pad/plot`, а також першу і останню точки.
- Прошити інструментаційний білд.
- Попросити користувача знову відкрити графік і надіслати серійний лог або фото разом з логом.
