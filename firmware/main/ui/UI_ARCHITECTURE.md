# Архітектура UI

Цей документ описує поточну архітектуру UI в прошивці, щоб уникнути плутанини в майбутньому.

## Активна система UI

- **Каталог**: `main/ui/screens/`
- **Опис**: Це основна і єдина система UI, яка має використовуватися для розробки. Вона містить реалізацію всіх екранів додатку.
- **Ключові файли**:
    - `screens.h`: Оголошує всі екрани (`ScreenId`), а також основні функції для керування ними (`screens_init`, `screens_show` і т.д.).
    - `screens.cpp`: Містить реалізацію функцій керування екранами та визначення глобальних змінних, таких як `screenRoots`.
    - `screen_*.cpp`: Кожен файл містить реалізацію одного конкретного екрану.

## Застарілі системи (ІГНОРУВАТИ)

Ці каталоги містять старий або автоматично згенерований код. **Не редагуйте файли в цих каталогах!**

1.  **`main/ui/picopixel/`**
    - **Опис**: Містить UI, згенерований за допомогою PicoPixel. Файли `ui.c` та `ui.h` з цього каталогу зараз використовуються як "прокладка" для виклику нової системи екранів з `main/ui/screens/`, але їх не слід змінювати.

2.  **`src/ui/`** (імовірно, в корені проекту або в `main/`)
    - **Опис**: Ще одна версія автоматично згенерованого UI. Повністю застаріла і не використовується.

## Як це працює

1.  `app_main.cpp` створює завдання `ui_task`.
2.  `ui_task` викликає `ui_backend_run()` з `lvgl_backend.cpp`.
3.  `lvgl_backend.cpp` викликає `ui_init()`.
4.  `ui_init()` (з `main/ui/picopixel/ui.c`) тепер викликає `screens_init()` з нашої нової системи.
5.  `screens_init()` створює всі екрани з каталогу `main/ui/screens/`.
